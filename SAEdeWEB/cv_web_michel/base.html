  • Durant ce projet de groupe, nous avons conçu un modèle de données pour une entreprise fictive. Nous avons commencé par élaborer un diagramme de classes pour définir les entités principales, leurs attributs et les relations entre elles. Lorsque certaines informations manquaient.

    À partir du diagramme de classes validé, nous avons transformé le modèle en schéma relationnel, créant les tables, les clés primaires et étrangères, et les contraintes nécessaires. Les requêtes SQL pour créer les tables ont été rédigées et exécutées sous Oracle (via ORCLIUT ou ORA2).

    Pour tester la base de données, nous avons inséré des données fictives pour valider les contraintes de clé, les relations référentielles et les types d'attributs. Nous avons utilisé des requêtes INSERT, UPDATE et DELETE, et consigné les résultats des tests, assurant ainsi le bon fonctionnement de la base. Ce projet a été une occasion de mettre en pratique nos connaissances théoriques tout en développant une base de données fonctionnelle.
